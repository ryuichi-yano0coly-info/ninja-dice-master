# Ninja Dice Master — 全画面プロトタイプ

`Readme/ndm_all_screens_prompts.md`（全7画面仕様）を軸に構築した、動作する Coin Master 風・和風ダイスゲームです。素材は image-gen（gpt-image-2 / low）で生成し、`remove_greenback.py` でグリーンバック（#00B140）を透過処理しています。React はローカル同梱・JSXは事前トランスパイル済みで、CDN 非依存・オフライン動作します。

## 実行方法

`file://` 直開きではなく、ローカルサーバー経由で開いてください。

```bash
cd prototypes/ninja-dice-master
python3 -m http.server 8777
# ブラウザで http://localhost:8777/ を開く（幅430pxのモバイル想定）
```

## 実装した7画面と遷移フロー

```
① メインロール ──roll──┬─ ぞろ目なし → コイン獲得（フロート表示）
                        ├─ Jackpot×2 → コンボ演出
                        └─ ぞろ目 → ② ぞろ目演出 ──┬─ Shield → メインへ（🛡️付与）
                                                     └─ 他 → ③ ボーナスロール ──┬─ Coin/Jackpot → 報酬 → メイン
                                                                                  ├─ Attack → ④ Attack選択 → ⑤ Attack結果 → メイン
                                                                                  └─ Steal → ⑥ Steal → メイン
① メインロール ⇄ ⑦ 城建設（TOPバーの🏯ボタンで行き来）
```

1. **メインロール** — 重み付きダイス、800msスピン→段階停止、獲得コインのフロート表示、キャラリアクション、オートロール（長押し）
2. **ぞろ目演出** — フラッシュ→BG_Bonus→出目別エフェクト（Attack/Steal/Shield/Jackpot/Coin）＋UI_Scrollバナー＋忍犬ジャンプ→自動遷移
3. **ボーナスロール** — 出目別テーブル（`BONUS_DICE_TABLES`）、ゴールドダイス回転、6タイルのハイライト
4. **Attack選択** — 相手の村の**建築物**（天守閣/蔵/石像/庭園、各レベルの段階画像で表示）に照準、タップで攻撃対象を決定
5. **Attack結果** — 成功/シールド防御の出し分け、コインカウントアップ、破壊演出
6. **Steal** — 4場所から3つ選択→順番にめくり→宝箱判定→倍率合算→受け取り
7. **村建設（コインマスター式）** — 天守閣・蔵・石像・庭園を**個別に**建設。ボタンを押すごとにレベルが上がり、建築物が段階的に完成へ近づく（各段階を専用画像で表示）。**全建築物は初期レベル0から開始**。全て完成でステージクリア→次のステージ（新しい村）

### 建築の段階画像
各建築物はレベルごとに専用の建築途中画像を持ちます（image-genで生成、透過処理済み）。
- 画像はステージテーマごとに `assets/images/building/<theme>/`（himeji/windsor/tajmahal/egypt/china/greece/aztec/russia/arabia/dragon）に格納
- 天守閣: `Castle_1〜4` → `Castle`（姫路のみ5段階、他テーマは`Castle_1〜2`→`Castle`の3段階）
- 蔵 / 石像 / 庭園: `Storehouse_1〜3` / `Statue_1〜3` / `Garden_1〜3`（各3段階、全テーマ共通）
- 建設ボタンでレベル+1、コスト `500 / 1,500 / 4,500 / 13,500 …`（`buildCost`）
- 攻撃画面は同じ建築物データ（`BUILD_ITEMS`）を流用し、相手の村の各建築物を攻撃対象にできます

## 状態管理・共通ルール（仕様準拠）

- 共有状態（coins/shields/stage/rolls/opponent）は `App` で一元管理し、各画面へ props とコールバックで受け渡し
- 画面遷移は React Router を使わず `App` の `screen` state による条件分岐（軽量SPA）
- カラーパレット・出目別カラー（`FACE_COLOR`）を全画面で統一
- コインのカウントアップは共通の `useCountUp` / `addCoins`（ease-out cubic）に集約
- 全画像は `assets/images/` の透過PNGを参照、読み込み失敗時は絵文字にフォールバック

## 3. 共通ルール・計算式

### 3.1 ダイスと役の抽選

- 出目5種: `coin`（重み2）/ `attack` / `steal` / `shield` / `jackpot`（各重み1）
- ダイス数は3/4/5個から選択（`diceCount`）。消費ロールは `DICE_COST = {3:3, 4:4, 5:5}`（振ったダイス数＝消費RP）
- **役ベース抽選**：各ダイスを独立に振らず、先に「役」を確率で決めてから出目n個（`decideHand(n)`）を構成する。オッズテーブル（`HAND_ODDS`）はダイス数ごとに異なる：

| 役 | 3個 | 4個 | 5個 |
|---|---|---|---|
| アタックぞろ目 | 5% | 5% | 5% |
| スティールぞろ目 | 5% | 5% | 5% |
| シールドぞろ目 | 5% | 5% | 5% |
| ジャックポットぞろ目 | 4% | 4% | 4% |
| コインぞろ目 | 5% | 5% | 5% |
| ジャックポット×2（レガシーコンボ） | 5% | 4% | 4% |
| **強襲（合体：attack×steal）** | — | 3% | 5% |
| **黄金律（合体：coin×jackpot）** | — | 3% | 5% |
| 通常役 | 残り約71% | 残り約66% | 残り約62% |

（通常役はいずれもぞろ目・JP2個以上・合体条件を満たさない組合せを重み付きで再抽選）

- **役判定（`evaluateHand`。唯一の真実。debug/demoもfacesを直接注入するだけでこの関数が正しく判定する）の優先度**：合体（`COMBO_DEFS`：役ごとの出目が2種とも2個以上同時に成立）＞ ぞろ目（同一出目3個以上）＞ ジャックポット×2（JP2個以上）＞ 通常
- 合体役の出目構成：強襲＝attack `ceil(n/2)` 個＋steal `floor(n/2)` 個、黄金律＝coin `ceil(n/2)` 個＋jackpot `floor(n/2)` 個（残りはシャッフル配置）

### 3.2 コイン獲得（通常役）

```
獲得コイン = stage × 100 + 500 × コイン出目数
```
- シールド出目が2個以上 → シールド+1（最大3）
- ジャックポット×2コンボ → 上式の獲得 + 3,000
- 通常ロールの10%（＋装備効果 cardDropBonus）でカード1枚ドロップ

### 3.3 ステージ基準額（ぞろ目報酬の元金）

```
coinBase(stage) = 250 × stage × (stage + 1)
（例: s1=500 / s3=3,000 / s5=7,500 / s10=27,500）
```

### 3.4 建設コスト

```
buildCost(level, stage) = round((33,000 + 22,000 × stage) × 1.85^level) × (1 − buildDiscount)
（例: stage3 Lv0≈99,000 / Lv3≈627,000）
```

### 3.5 積むダイス数（diceCount）

3個／4個／5個から選択（画面下の🎲ボタンで循環）。ロール消費は `DICE_COST = {3:3, 4:4, 5:5}`（振ったダイス数＝消費RP）。ぞろ目役の効果は成立面数 k（=ダイス数）でスケールする（スティール棟数・アタック破壊数=k−2・JP倍率×(k−2)・コインボーナス表格上げ・シールド超過コイン変換）。
**報酬倍率としての×betは廃止**（旧仕様）。ダイス数を増やすメリットは、4個以上でのみ合体役（強襲・黄金律）が抽選対象に入ること。

## 4. 画面別仕様

### 4.0 ローディング画面

- 全画像リスト（`manifest.js` の `NDM_IMAGES`）を非同期プリロードし、読み込み割合を進捗バー表示。
- 100%到達後もチラつき防止で最低約1.1秒はスプラッシュ表示。安全タイムアウト90秒。

### 4.1 メインロール画面（main）

**UI構成**
- TOPバー：コイン残高（＋でショップ）/ シールド3ドット / サウンドON・OFF / 昼夜切替 / メニュー
- 左サイドレール：村建設・仲間・カード・討伐（チケット数バッジ）
- 右サイドレール：商店・催事（シーズン）・招待
- 対戦相手カード：顔・名前・保有コイン
- ダイス3〜5個（演出3種：3D立体＝既定 / 下から飛ばし / 回転）
- 獲得バナー（今回の獲得 +N）
- 残りロールバー（現在/最大・回復タイマー表示・「🎬無料+5」ボタン＝広告想定でロール+5）
- ダイス数選択（🎲3個/4個/5個を循環。消費ロールは1/2/3。4個以上で合体役が抽選対象に入る）
- 大ロールボタン（長押し550ms以上でオートロール開始。オート中は「■停止」に変化）
- 装備中キャラのマスコット（左下・6フレームループアニメ・タップで仲間画面）
- 常時演出：桜吹雪パーティクル13枚
- デバッグFAB（🐞）：役の強制指定・演出切替・ショップ再ロール等

**ロール処理フロー**
1. `rolls < rollCost` なら「ロールが足りません」→中断（オート中は解除）
2. ロール消費＝`rollCost`（装備効果 freeRollChance の確率で無料）。消費数はシーズンXPに加算
3. 役を抽選（3.1、`decideHand(diceCount)`）→ダイス演出
   - 2D演出：80ms間隔で面を高速切替、520msから160msずつずらして1個ずつ停止（n個目は520+(n-1)×160ms）、その60ms後に判定
   - 3D演出：立方体が結果面へ転がって着地（発射は150msずつスタガー）、1,700msで判定
4. 判定（`evaluateHand`。3.1の優先度）：
   - **合体（強襲/黄金律）** → 強襲はassault画面へ即遷移（ロール解放タイマーなし＝画面遷移で解放）。黄金律はメインに残るため1.3秒後にロール解放してからジャックポットスロットへ合流
   - **ぞろ目** → Zorume演出へ委譲（1.3秒後にロール解放）
   - **JP×2コンボ** → 獲得+3,000、ジャックポットエフェクト
   - **通常** → コイン獲得（3.2）、コイン噴き上げパーティクル（約1.35秒、終了で次ロール可能に）、シールド2個で+1、10%でカードドロップ（カードは一瞬表示→左レールの「カード」ボタンへ飛翔）
5. オートロール：ロール解放から350ms後に次ロールを自動実行。演出オーバーレイ中は一時停止

### 4.2 ぞろ目演出（Zorumeオーバーレイ）

- タイムライン：フラッシュ(0.22s)→本体表示(〜0.9s)→遷移表示(〜1.3s)→自動で次へ
- 表示物：出目別背景（attack=BG_Attack / steal=BG_Steal / 他=BG_Bonus）、出目別エフェクト画像、ダイス3個（80msずつ遅延ポップ）、巻物バナー（「ぞろ目！！」or「JACKPOT!!」＋出目バッジ）、忍犬ジャンプ
- 完了後のルーティング：coin→bonus / attack→attackSelect（アタックボーナスを内部で先に抽選）/ steal→steal / jackpot→ジャックポットスロット / shield→シールド獲得演出

### 4.3 ボーナスロール画面（bonus）※小判ぞろ目時

- 金色の3Dダイス1個。入場0.5秒後に自動でロール、1.65秒後に結果表示
- テーブル（`BONUS_DICE_TABLES.coin`）：×2 / ×5 / ×10 / ×20 / ×50 / ×100（6面均等抽選）
- 画面には「倍率の元金 = coinBase(stage)」を最初から表示。結果時に式（元金 × 倍率）とカウントアップ表示＋コインパーティクル
- **獲得** = `coinBase × multiplier × coinMult`（coinMult=装備効果）
- 「次へ」でメインへ。**40%の確率でレイドチケット+1**

### 4.4 アタック選択画面（attackSelect）

- 相手の村（建設画面と同レイアウト・テーマは現ステージ準拠、レベル固定：天守閣3/蔵2/石像2/庭園1）を表示し、任意の建物1つをタップして攻撃
- 事前にアタックボーナス（`BONUS_DICE_TABLES.attack` から1面）を抽選済み：

| 面 | 名称 | damage（破壊棟数） | coinRate（獲得率） |
|---|---|---|---|
| 1 | 一撃 | 1 | 0% |
| 2 | 強奪 | 0 | 10% |
| 3 | 襲撃 | 1 | 5% |
| 4 | 連撃 | 2 | 0% |
| 5 | 火攻め | 2 | 15% |
| 6 | 城壊し | 3 | 25% |

- **成否判定（確定的）**：相手の `shields > 0` かつ装備効果 `ignoreShield` でない場合は防御される。防御時は相手シールド−1
- 成功演出：インパクト連番(8f×55ms)→コイン噴出連番(8f×95ms)→建物を破壊状態（瓦礫）表示→結果画面へ
- 防御演出：シールドエフェクト(0.76s)→結果画面へ
- 予測パネル表示：成功時=相手コインの約 coinRate%（既定25%）、防御時=約7%
- 破壊棟数＝成立面数k−2（k=4で2棟、k=5で3棟。タップ棟＋価値上位棟を自動巻き込み）。コインは `×ATTACK_COIN_KMULT{3:1,4:1.5,5:2}`、相手コインの50%が上限
- キャンセル不可（「逃げる」「←戻る」ボタンは廃止。必ず攻撃を完了する）

### 4.5 アタック結果画面（attackResult）

- 成功：「攻撃成功！！」＋壊れた城＋破壊棟数（damage）表示。防御：「シールドに阻まれた！」＋盾エフェクト
- **獲得コイン**：
  - 成功 = `floor(相手コイン × coinRate × attackMult)`
  - 防御 = `floor(相手コイン × 0.07 × attackMult)`
- コインはカウントアップ表示（1秒）。「次へ」で加算→対戦相手ローテーション→メインへ

### 4.5B 強襲画面（assault）※合体役「強襲」（attack×steal）成立時

- ダイス4個以上で「アタック2個以上＋スティール2個以上」が同時成立した時のみ出現する専用画面（成立時はメイン画面で約1.3秒の合体カットイン→遷移）。攻撃とスティール両方の効果が発動し、合計に合体ボーナス×1.15（`COMBO_K`）が掛かる
- **3タップ選択制**（スティール画面と同じUX）：相手の村で建物を3回タップ。1タップ＝その棟を即座に「破壊＋略奪」（確認ゲートなし・キャンセル不可）
  - 入場時に確定：アタックボーナス1面抽選（coinRate用）、`aFull = floor(相手コイン × coinRate × attackMult)`、相手シールド枚数スナップショット
  - 各棟の略奪額はタップ時に抽選（`stealFromBuilding`、スティール画面と同じタイミング）
  - **シールドはタップごとに1枚消費**：防がれたタップは破壊なし・略奪×0.6（`ASSAULT_SHIELD_STEAL_DECAY`）・追い銭の通過数に数えない。`ignoreShield`装備は全通し。相手シールドは最大2なので3タップ目は必ずフル
  - `S = Σタップ3棟の略奪額（減衰込み）`、`A_eff = round(aFull × フルで通ったタップ数 / 3)`
  - `total = round((S + A_eff) × 1.15)`
- 3タップ完了→サマリー（約1秒カウントアップ、`ASSAULT_SUMMARY_HOLD=2200ms`で自動受取。タップで即スキップ可）
- 受取時：コイン加算・消費したシールド分を相手から減算・対戦相手ローテーション（1回のみ）→メインへ。オートロール中は本画面でタップ待ちとなり、受取後にオートが自動再開する

### 4.6 スティール画面（steal）

- 相手の村（アタックと同じ見た目）から建物をタップして盗む（進捗●○○表示）。タップ数は成立面数kに応じて `STEAL_TAPS_BY_K = {3:3, 4:4, 5:4}`（k=5はさらに総額×1.2の5面ボーナス）
- 建物別パラメータ（`STEAL_BUILDING`）：

| 建物 | コイン範囲 | 宝箱確率 |
|---|---|---|
| 天守閣 | 40,000〜85,000 | 15% |
| 蔵 | 50,000〜90,000 | 38% |
| 石像 | 15,000〜35,000 | 50% |
| 庭園 | 25,000〜55,000 | 22% |

- タップ→盗み連番演出(7f×80ms)→金額確定・✓表示。宝箱が出たら「宝箱GET！」演出(1秒)
- 装備効果 `stealLastSpot`（影のくノ一）：3つ盗んだ後、残り1か所も自動で強奪
- 3件（+自動1件）完了でサマリー表示：
  - **合計 = round(小計 × stealMult)**（内訳行つき、1秒カウントアップ）
  - 宝箱の中身：1箱につき **50%で仲間のかけら／50%でカード**（同時には出ない）
    - かけら：解放済みキャラから未コンプ優先で抽選。枚数はレジェンド5〜10、他10〜20 ×(1+pieceBonus)
    - カード：GOLD率25%で抽選
- 「受け取る！」→コイン加算・宝箱報酬付与→相手ローテーション→メイン（「逃げる」ボタンは廃止。必ず完走する）

### 4.7 村建設画面（castle）

### 4.10 討伐戦（clan）／討伐編成（raidParty）

- ボス10体（村ステージテーマと1:1対応）。HPは%制（100→0）
- **攻撃**：レイドチケット1枚消費・編成1体以上必須

  - **ダメージ% = clamp(round((partyAP + 6) / (2.5 + (boss−1) × 1.4)), 3, 60)**
- **マイルストン報酬**：HP75/50/25%を初めて割った時、ボス撃破報酬の8/10/12%を即時付与

- 全10体撃破で「討伐完了」表示

- **討伐編成画面**：所持済みキャラから最大4体選択（タップでトグル、選択順バッジ表示）。合計攻撃力を表示。localStorage 永続
